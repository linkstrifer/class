HTML

HTML5 es la actualización de HTML, pero también es un término de marketing para agrupar las nuevas tecnologías de desarrollo de aplicaciones web.

HTML5 - Actualización de HTML

HTML son las siglas de HyperText Markup Language, que traduce lenguaje de marcas de hipertexto, es el lenguaje con que se modelan las paginas de Internet.

HTML5 es la quinta revisión importante del lenguaje básico de la World Wide Web. Wikipedia

HTML5 establece una serie de nuevos elementos y atributos. Algunos de ellos son técnicamente similares a algunas etiquetas pero tienen un significado semántico. Otros elementos proporcionan nuevas funcionalidades.

Además algunos elementos quedan obsoletos.

HTML5 - Termino de marketing

Es una colección de herramientas y estandares para el desarrollo web.

3D, gráficos y efectos, conectividad, CSS3, acceso a dispositivos, multimedia, almacenamiento offline, rendimiento, semántica.

HTML5 - 3D, gráficos y efectos

La Web ha sido siempre un medio visual, pero restringido en el mejor de los casos. Hasta hace poco, los desarrolladores de HTML solo podían utilizar CSS y JavaScript para crear animaciones o efectos visuales para sus sitios web, o tenían que recurrir a complementos como Flash. HTML5Rocks

Sin embargo, esta situación ha cambiado con la incorporación de tecnologías como el elemento canvas, la especificación WebGL y las imágenes SVG. De hecho, hay muchas funciones nuevas que permiten crear gráficos en la Web: Canvas 2D, WebGL, SVG, las transformaciones 3D de CSS y ​​el estándar SMIL.

HTML5 - Conectividad

Conectividad más eficiente significa disponer de chats en tiempo real, juegos más rápidos y una mejor comunicación. Los Web Sockets y los Server-Sent Events enviados por el servidor transmitirán datos entre cliente y servidor de una forma mucho más eficiente que antes. W3C

HTML5 - CSS3

CSS3 ofrece un amplio rango de estilos y efectos, mejorando las aplicaciones web sin sacrificar el rendimiento o la estructura semántica. Además, el formato de fuentes web abiertas (WOFF) ofrece flexibilidad tipográfica y control más allá de lo que la web ha ofrecido hasta ahora. W3C

HTML5 - Acceso a dispositivos

La geo-localización es sólo el comienzo. HTML5 ayudará a las aplicaciones a acceder a los dispositivos más allá del navegador. W3C

HTML5 - Multimedia

Audio y vídeo son los elementos más destacados en la web del HTML5, conviviendo en harmonía con tus aplicaciones y sitios. W3C

HTML5 - Almacenamiento offline

Las aplicaciones de HTML5 se iniciarán más rápido y funcionarán incluso si no se dispone de conexión a internet, gracias a la Caché de Aplicaciones, Almacenamiento Local, DB Indexadas y las APIs de ficheros. W3C

HTML5 - Rendimiento

Los navegadores modernos pueden ofrecer sitios y aplicaciones que compiten con el rendimiento de las nativas en todo tipo de dispositivos y pantallas. Este rendimiento no viene gratis, como desarrolladores tenemos que usar una variedad de técnicas y tecnologías para sacar el máximo provecho del navegador. El rendimiento es una parte fundamental de la experiencia del usuario, es un tema constante en todos los aspectos del desarrollo. HTML5Rocks

HTML5 - Semántica

Ofreciendo significado a la estructura, la semántica es la parte central y frontal del HTML5. Un conjunto de etiquetas más ricas, con RDFa, microdatos y microformatos, harán que la web basada en datos sea más útil para programas y usuarios. W3C

Sintaxis HTML

HTML esta basado en etiquetas y atributos.

Cada etiqueta hace referencia a contenido del documento.

Las etiquetas son palabras dentro de paréntesis angulares < y >. Las etiquetas normalmente vienen en pares, una etiqueta de inicio y una etiqueta de fin o cierre.

Ejemplo:

			<etiqueta> </etiqueta>

La etiqueta de cierre es similar que la de inicio pero con una barra inclinada antes del nombre de la etiqueta.

Sintaxis HTML - Elementos HTML

"Etiqueta HTML" y "elemento HTML" son dos cosas diferentes.

Un elemento HTML es todo lo que esta entre las etiquetas HTML, incluyendo las etiquetas.

Ejemplo de elemento HTML:

			<p>Esto es un parrafo</p>

Sintaxis HTML - Atributos HTML

Los elementos HTML pueden tener atributos. Los atributos proporcionan información adicional sobre el elemento.

Son palabras que van dentro de la etiqueta de inicio.
La sintaxis de un atributo es nombre="valor"

Ejemplo:

			<a href="url">Enlace</a>

La etiqueta a se utiliza para crear un enlace. El atributo href especifica el destino del enlace.

Sintaxis HTML - Estructura de una pagina

Todas las paginas del mundo están conformadas por 4 etiquetas básicas:

<!DOCTYPE html>, <html>, <head> y <body>

Sintaxis HTML - La etiqueta <!DOCTYPE html>

La etiqueta <!DOCTYPE html> ayuda al navegador a mostar correctamente el contenido de la pagina, en esta etiqueta se le dice al navegador que versión de HTML se esta usando.

HTML 4.01 <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
"http://www.w3.org/TR/html4/loose.dtd">

HTML5 <!DOCTYPE html>

Es mucho más fácil y cool usar la etiqueta de HTML5.

Sintaxis HTML - La etiqueta <html>

La etiqueta <html> le dice al navegador que es un documento html, también le dice que esta es la raíz del documento.

Esta etiqueta es el contenedor de todos los elementos HTML, a excepción del <!DOCTYPE html>

Sintaxis HTML - La etiqueta <head>

La etiqueta <html> es la cabecera del documento, contiene el título, y puede contener scripts, hojas de estilos, meta-información y otras cosas.

Estos son los elementos que pueden ir dentro de la cabecera del documento:
<title>, <style>, <base>, <link>, <meta>, <script>, <noscript>.

Sintaxis HTML - La etiqueta <body>

La etiqueta <body> define el cuerpo del documento.

Aquí es donde se encuentran todos los elementos como los parrafos, enlaces, imágenes, videos, tablas, listas ... en general, todo el contenido visible.

Sintaxis HTML - Estructura de una pagina

Estructura de una pagina HTML sin contenido.

				
<!DOCTYPE html>
<html>
  <head>
    <title></title>
  </head>
  <body>
  </body>
</html>

La etiqueta <title> es requerida en la cabecera. Esta etiqueta define el título de la pagina.

Proyecto

Para que no sea simplemente teoría, crearemos paso a paso este sitio.

Nota

Para empezar el proyecto explicaré el HTML base e iremos agregando lo que vayamos necesitando para no hacerlo tan difícil de entender al principio.

HTML básico

Primero necesitamos el html básico de cualquier sitio web.

				
<!doctype html>
<html>
  <head>
    <title>¿Estos son los droides que buscas?</title>
  </head>
  <body>
  </body>
</html>

Como pueden ver el título de nuestro proyecto es ¿Estos son los droides que buscas?

Charset

Ya tenemos el código básico, ahora vamos a añadirle soporte para carácteres especiales como las tildes o la letra ñ.

Simplemente tenemos que agregar una etiqueta al <head> de nuestro sitio.

				
<meta charset="utf-8">

Si no hacemos esto en vez de letras con tildes encontraremos codigos como &aacute; en lugar de á.

Mapa

Ahora necesitamos un espacio para cargar el mapa.

Para esto usaremos una etiqueta <div> en el <body> de nuestro sitio con el identificador map.

				
<div id="map">

La etiqueta <div> se usa para definir una división o sección en el documento.

El atributo id se usa como identificador dentro del documento, debe ser único en la pagina, se utiliza cuando se necesita una manera de buscar un elemento único, es decir, que no se repita.

Cabecera de la pagina

Ahora agregaremos la cabecera de la página, que es diferente a la cabecera del documento.

La cabecera de la página se usa para que el usuario vea de que se trata nuestro sitio, algo así como una introducción, mientras que la cabecera del documento es donde estan los estilos, scripts, etiquetas meta.

Para la cabecera de la página usaremos una etiqueta <header> introducida en html5 la cual tiene gran valor semántico, será lo primero que los buscadores lean dentro de la pagina, así que vamos a poner un título dentro de la cabecera.

				
<header>
  <h1>¿Estos son los droides que buscas?</h1>
</header>

Para los títulos se usan etiquetas especiales que van por niveles de jerarquía desde el 1 al 6 siendo el 1 el de mayor jerarquía. Las etiquetas son <h1> <h2> <h3> <h4> <h5> <h6>.

La etiqueta <h1> al ser la de mayor jerarquía no debería repetirse en el documento para no perder su valor semántico.

Galeria de fotos

Ya casi terminamos, ahora nos falta la sección donde se cargarán las fotos.

Para esta parte usarémos un <div> con el identificador gallery.

				
<div id="gallery"></div>

No hay más explicación, practicamente es igual a la sección del mapa.

Mensaje de error

Y para terminar necesitamos un mensaje que aparezca cuando la foto no tiene el lugar donde fue tomada.

Para esto usaremos otro <div> con el identificador not-map.

				
<div id="not-map"></div>
